Implementing a Simple Stack in C++

Feb 29, 2024· Tutorial Island

Introduction

Today we are going to implement a simple stack data structure in C++ using an array as the underlying storage. A stack is a fundamental data structure that follows the Last-In, First-Out (LIFO) principle. This means the last element added to the stack is the first element removed. We will implement the core operations of a stack: push (add an element), pop (remove an element), and peek (view the top element).

Explanation

Implementing a stack might seem straightforward, but several challenges need to be addressed.

  1. Array Management: We’ll use an array to store the stack elements. We need to decide how to handle the size of the array. For simplicity, we’ll use a fixed-size array, but in a real-world scenario, a dynamic array (like std::vector) would be preferred for automatic resizing. This fixed size introduces the possibility of stack overflow if we try to push more elements than the array can hold.

  2. Top Pointer: We need a way to track the top element of the stack. We can use an integer variable, often named top, which represents the index of the topmost element in the array. Initially, top will be set to -1, indicating an empty stack.

  3. Empty Stack Handling: When attempting to pop or peek from an empty stack, we need to handle this gracefully to prevent errors. We’ll throw an exception in these cases.

  4. Error Handling: While we chose to throw errors, a better approach could involve returning a status code, an std::optional, or implementing a custom exception class for improved error reporting.

To implement this, we will use the following C++ features:

  • Classes: To encapsulate the stack’s data (the array and the top index) and its operations.
  • Arrays: To store the stack elements.
  • Exceptions: To handle error conditions like stack overflow and underflow.
  • Asserts: To verify the correctness of our functions during development and testing.

Requirements

Here’s a breakdown of the functional requirements for our stack:

  • Push: Adds an element to the top of the stack. The top index must be incremented. If the stack is full, throw an exception.
  • Pop: Removes the element from the top of the stack and returns it. The top index must be decremented. If the stack is empty, throw an exception.
  • Peek: Returns the element at the top of the stack without removing it. If the stack is empty, throw an exception.
  • IsEmpty: Returns true if the stack is empty, false otherwise.

Solutions

Here are code snippets demonstrating the implementation of each functional requirement:

#include <iostream>
#include <cassert>
#include <stdexcept>

class Stack {
private:
    int* arr;
    int top;
    int capacity;

public:
    Stack(int size) : capacity(size), top(-1) {
        arr = new int[capacity];
    }

    ~Stack() {
        delete[] arr;
    }

    void push(int value) {
        if (top >= capacity - 1) {
            throw std::overflow_error("Stack Overflow");
        }
        arr[++top] = value;
    }

This code block defines the push operation. It first checks if the stack is full. If it is, an std::overflow_error is thrown. Otherwise, it increments the top index and adds the new value to the array at the new top position. We choose a simple std::overflow_error because it’s built-in and easy to use for demonstration purposes. A more robust implementation might define a custom exception type.

    int pop() {
        if (top < 0) {
            throw std::underflow_error("Stack Underflow");
        }
        return arr[top--];
    }

This code block defines the pop operation. It first checks if the stack is empty. If it is, an std::underflow_error is thrown. Otherwise, it retrieves the value at the current top position, decrements the top index, and returns the retrieved value. Like with push, a more descriptive custom exception could be used.

    int peek() {
        if (top < 0) {
            throw std::underflow_error("Stack is empty");
        }
        return arr[top];
    }

This code block defines the peek operation. It checks if the stack is empty and throws an exception if it is. Otherwise, it returns the value at the current top position without modifying the top index.

    bool isEmpty() const {
        return top == -1;
    }
};

This code block defines the isEmpty operation. It simply returns true if the top index is -1 (indicating an empty stack) and false otherwise.

Thoughts

Let’s combine all the code snippets into a single, complete implementation of the stack data structure, along with some basic tests using assert:

#include <iostream>
#include <cassert>
#include <stdexcept>

class Stack {
private:
    int* arr;
    int top;
    int capacity;

public:
    Stack(int size) : capacity(size), top(-1) {
        arr = new int[capacity];
    }

    ~Stack() {
        delete[] arr;
    }

    void push(int value) {
        if (top >= capacity - 1) {
            throw std::overflow_error("Stack Overflow");
        }
        arr[++top] = value;
    }

    int pop() {
        if (top < 0) {
            throw std::underflow_error("Stack Underflow");
        }
        return arr[top--];
    }

    int peek() {
        if (top < 0) {
            throw std::underflow_error("Stack is empty");
        }
        return arr[top];
    }

    bool isEmpty() const {
        return top == -1;
    }
};

int main() {
    Stack myStack(5);

    assert(myStack.isEmpty() == true);

    myStack.push(10);
    myStack.push(20);
    myStack.push(30);

    assert(myStack.peek() == 30);
    assert(myStack.isEmpty() == false);

    assert(myStack.pop() == 30);
    assert(myStack.pop() == 20);
    assert(myStack.pop() == 10);

    assert(myStack.isEmpty() == true);

    bool exceptionThrown = false;
    try {
        myStack.pop(); // Should throw an exception
    } catch (const std::underflow_error& e) {
        exceptionThrown = true;
    }
    assert(exceptionThrown == true);


    std::cout << "All tests passed!" << std::endl;

    return 0;
}

This complete example demonstrates the core functionality of a stack using a fixed-size array. Remember that in real-world applications, using std::vector or other dynamic allocation techniques is generally preferred to handle stack resizing automatically. The assert statements at the end verify the basic correctness of our implementation, confirming that the push, pop, peek, and isEmpty operations behave as expected. We also included an assert to make sure that the pop method throws an error as intended.
