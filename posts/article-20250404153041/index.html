Building a Simple Text-Based Adventure Game in C++

Introduction

Today, we’re going to embark on a journey to create a rudimentary text-based adventure game using C++. Imagine a scenario where the player is lost in a dark forest and needs to find their way out. The player will be presented with choices at each step, and their decisions will determine their fate. This project will introduce you to fundamental C++ concepts in a fun and engaging way.

Explanation

Developing a text-based adventure game, even a simple one, presents several challenges:

  1. Handling User Input: We need a way to receive and process the player’s choices. This involves reading input from the console and interpreting it.
  2. Conditional Logic: The game’s story branches based on player choices. We’ll need to use conditional statements (if, else if, else) to determine the next course of action.
  3. Game State Management: We need to keep track of the player’s current location, inventory (if any), and other relevant information.
  4. Outputting Text: The game needs to display descriptions, options, and feedback to the player.

To address these challenges, we’ll be using the following C++ functionalities:

  • iostream: For input and output operations (reading player choices and displaying text).
  • string: For handling text-based input and descriptions.
  • vector (optionally, for more complex scenarios): For managing lists of items or locations.
  • if, else if, else: For creating the branching narrative logic.
  • Functions: To modularize our code and make it more readable.

Requirements

Functional Requirements:

  1. The game should present the player with a starting scenario (e.g., “You are in a dark forest…”).
  2. The game should offer the player a set of choices (e.g., “Go north”, “Go east”).
  3. The game should read and interpret the player’s choice from the console.
  4. Based on the player’s choice, the game should:
    • Update the game state (e.g., move the player to a new location).
    • Display a new scenario and choices.
    • End the game if the player reaches a winning or losing condition.
  5. The game should handle invalid input gracefully (e.g., if the player enters a choice that is not available).
  6. The game should use at least 3 different scenarios.

Non-Functional Requirements:

  1. The code should be well-structured and readable.
  2. The code should be commented to explain its functionality.
  3. Each function should have assertions to validate its behavior.

Solutions

#include <iostream>
#include <string>
#include <cassert>

// Function to display a scenario and get the player's choice
int getPlayerChoice(const std::string& scenario, const std::string& option1, const std::string& option2) {
    std::cout << scenario << std::endl;
    std::cout << "1. " << option1 << std::endl;
    std::cout << "2. " << option2 << std::endl;
    std::cout << "Enter your choice (1 or 2): ";

    int choice;
    std::cin >> choice;

    // Validate input
    assert(choice == 1 || choice == 2); // Ensure the choice is valid

    return choice;
}

// Function to handle the first scenario (forest entrance)
std::string handleForestEntrance(int choice) {
    if (choice == 1) {
        return "You bravely head deeper into the forest...";
    } else {
        return "You cautiously turn back and search for another path...";
    }
}

Explanation:

  • getPlayerChoice: This function takes a scenario description and two options as input. It displays them to the player and prompts for a choice. It then reads the input from the console and returns the player’s choice as an integer. The assertion makes sure that the player’s choice is between 1 and 2. We need the function because all the scenarios will require us to show options and recover user input.
  • handleForestEntrance: This function takes the player’s choice from the first scenario and returns a string describing the outcome of that choice. If the player chooses 1, they go deeper into the forest. If they choose 2, they turn back. assert(handleForestEntrance(1) == “You bravely head deeper into the forest…”); assert(handleForestEntrance(2) == “You cautiously turn back and search for another path…”);

## Thoughts

Okay, let's pull everything together into a complete text-based adventure game.  We'll use the functions we've defined to create a simple story with three branching points.

```c++
#include <iostream>
#include <string>
#include <cassert>

// Function to display a scenario and get the player's choice
int getPlayerChoice(const std::string& scenario, const std::string& option1, const std::string& option2) {
    std::cout << scenario << std::endl;
    std::cout << "1. " << option1 << std::endl;
    std::cout << "2. " << option2 << std::endl;
    std::cout << "Enter your choice (1 or 2): ";

    int choice;
    std::cin >> choice;

    // Validate input
    return choice;
}

// Function to handle the first scenario (forest entrance)
std::string handleForestEntrance(int choice) {
    if (choice == 1) {
        return "You bravely head deeper into the forest...";
    } else {
        return "You cautiously turn back and search for another path...";
    }
}

// Function to handle the second scenario (river crossing)
std::string handleRiverCrossing(int choice) {
    if (choice == 1) {
        return "You attempt to swim across the raging river and barely make it to the other side...";
    } else {
        return "You search for a safer way to cross and discover a hidden bridge...";
    }
}

// Function to handle the third scenario (cave entrance)
std::string handleCaveEntrance(int choice) {
    if (choice == 1) {
        return "You bravely venture into the dark cave...";
    } else {
        return "You decide the cave looks too dangerous and continue on your path...";
    }
}

int main() {
    std::string current_scenario;

    // Starting Scenario
    std::cout << "You are lost in a dark forest. You see a path leading deeper into the woods, and another leading back the way you came." << std::endl;
    int choice1 = getPlayerChoice("What do you do?", "Go deeper into the forest", "Turn back");
    current_scenario = handleForestEntrance(choice1);
    std::cout << current_scenario << std::endl;

    // River Scenario
    std::cout << "You come to a wide, raging river.  You could try to swim across, or search for another way." << std::endl;
    int choice2 = getPlayerChoice("What do you do?", "Swim across", "Search for a bridge");
    current_scenario = handleRiverCrossing(choice2);
    std::cout << current_scenario << std::endl;

    // Cave Scenario
    std::cout << "You stumble upon the entrance to a dark cave.  Do you dare enter, or continue on your path?" << std::endl;
    int choice3 = getPlayerChoice("What do you do?", "Enter the cave", "Continue on the path");
    current_scenario = handleCaveEntrance(choice3);
    std::cout << current_scenario << std::endl;

    std::cout << "Congratulations! You have survived your adventure... for now." << std::endl;

    return 0;
}

// Assertions
#include <cassert>
int main() {
    assert(handleForestEntrance(1) == "You bravely head deeper into the forest...");
    assert(handleForestEntrance(2) == "You cautiously turn back and search for another path...");
    assert(handleRiverCrossing(1) == "You attempt to swim across the raging river and barely make it to the other side...");
    assert(handleRiverCrossing(2) == "You search for a safer way to cross and discover a hidden bridge...");
    assert(handleCaveEntrance(1) == "You bravely venture into the dark cave...");
    assert(handleCaveEntrance(2) == "You decide the cave looks too dangerous and continue on your path...");
    std::cout << "Assertions passed!" << std::endl;
}

Explanation:

  • The main function orchestrates the game’s flow.
  • It presents each scenario using std::cout and then uses getPlayerChoice to get the player’s input.
  • The handle... functions are called to determine the outcome of each choice.
  • The game ends with a concluding message.

This is a very basic example, but it demonstrates the core principles of creating a text-based adventure game. You can expand on this by adding more scenarios, items, and more complex logic. Good luck, adventurer!
