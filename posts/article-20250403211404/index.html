Haskell - Implementing a Simple Caesar Cipher

Jan 24, 2024· Tutorial Island

Introduction

Today, we’re going to implement a classic cryptographic algorithm: the Caesar cipher. This cipher shifts each letter in a message by a fixed number of positions down the alphabet. While not secure, it provides a great way to learn about character manipulation and basic functional programming concepts in Haskell. We’ll take a plain text message and encrypt it using a key (the shift value), and also be able to decrypt back to the original message.

Explanation

The Caesar cipher presents a few key challenges:

  1. Character Representation: We need to understand how characters are represented in Haskell (Unicode) and how to manipulate them.
  2. Shifting: We need a way to shift each character by a given amount, wrapping around the alphabet if necessary. For example, shifting ‘z’ by 1 should result in ‘a’.
  3. Case Handling: We need to decide whether to preserve the case of the original letters (uppercase remains uppercase, lowercase remains lowercase) or convert everything to a single case. For this tutorial, we will preserve the case.
  4. Non-alphabetic characters: We should handle characters that are not letters (e.g., spaces, punctuation) gracefully, leaving them unchanged.

To address these challenges, we’ll use the following Haskell features:

  • Data.Char: This module provides functions for character manipulation, such as checking if a character is a letter (isLetter), converting between characters and their ASCII/Unicode representation (ord, chr), and changing case (toLower, toUpper).
  • mod operator: This operator will be crucial for wrapping around the alphabet. x mod n gives the remainder of the division of x by n.
  • List Comprehensions: We’ll use list comprehensions to efficiently process each character in the input string.
  • Function Composition: We can make our code more readable and concise by composing functions.

Requirements

  • Functional Requirement 1: Implement an encrypt function that takes a shift value (integer) and a string as input, and returns the Caesar-encrypted string.
  • Functional Requirement 2: Implement a decrypt function that takes a shift value (integer) and an encrypted string as input, and returns the original string. Decryption is simply encryption with the negative of the shift value.
  • Functional Requirement 3: Preserve the case of letters during encryption and decryption.
  • Functional Requirement 4: Leave non-alphabetic characters unchanged during encryption and decryption.
  • Non-Functional Requirement: The code should be readable and well-documented.

Solutions

import Data.Char (isLetter, ord, chr)

-- | Shifts a single character by a given amount, wrapping around the alphabet.
-- | Preserves the case of the character.
shiftChar :: Int -> Char -> Char
shiftChar shift char
  | isLetter char =
      let base = if isUpper char then ord 'A' else ord 'a'
          shifted = (ord char - base + shift) `mod` 26
      in chr (base + shifted)
  | otherwise = char

-- | Checks if a character is uppercase.
isUpper :: Char -> Bool
isUpper c = c >= 'A' && c <= 'Z'

main :: IO ()
main = do
  putStrLn "Shift 'a' by 1:"
  print (shiftChar 1 'a')
  putStrLn "Shift 'z' by 1:"
  print (shiftChar 1 'z')

This shiftChar function takes a shift value and a character as input. It checks if the character is a letter. If it is, it calculates the shifted character code, wrapping around the alphabet using the mod operator. If the character is not a letter, it returns the character unchanged. The isUpper function simply checks if the character is within the range of uppercase letters (A-Z). We are ensuring that ‘a’ shifted by 1 is ‘b’, and ‘z’ shifted by 1 goes back to ‘a’, demonstrating the wrap-around behavior.

  putStrLn "Encrypt \"Hello, World!\" with shift 5:"
  print (encrypt 5 "Hello, World!")

The encrypt function uses a list comprehension to apply the shiftChar function to each character in the input string. This efficiently creates the encrypted string. We test it with “abc” shifted by 3, which should result in “def”. We also test “Hello, World!” which includes uppercase letters, spaces, and punctuation to confirm that all of those are handled correctly.

encrypt :: Int -> String -> String
encrypt shift str = [shiftChar shift c | c <- str]

-- | Decrypts a string using the Caesar cipher.
decrypt :: Int -> String -> String
decrypt shift str = encrypt (-shift) str

main :: IO ()
main = do
  putStrLn "Decrypt \"def\" with shift 3:"
  print (decrypt 3 "def")
  putStrLn "Decrypt \"Mjqqt, Btwqi!\" with shift 5:"
  print (decrypt 5 "Mjqqt, Btwqi!")

The decrypt function leverages the encrypt function, negating the shift value. Since shifting by -n is the inverse operation of shifting by n, this effectively decrypts the string. We are testing by decrypting “def” shifted by 3 and “Mjqqt, Btwqi!” shifted by 5 to make sure the original words come back out.

  let encrypted = encrypt 3 "Hello, World!"

  let decrypted = decrypt 3 encrypted

  print ("Encrypted2: " ++ encrypted2)
  let decrypted2 = decrypt 13 encrypted2
  print ("Decrypted2: " ++ decrypted2)
