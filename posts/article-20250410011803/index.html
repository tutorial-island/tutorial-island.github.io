TypeScript Generics: Building a Reusable Data Mapper

Oct 27, 2023· Tutorial Island

Introduction

Today, we’re going to build a reusable data mapper using TypeScript generics. Imagine you have different data structures coming from various APIs or databases, and you need to transform them into a unified, consistent format for your application. Writing a specific mapper for each data structure would be tedious and error-prone. Generics will allow us to create a flexible mapper that can handle various data types while maintaining type safety.

Explanation

The main challenge lies in creating a function that can accept different input data types and transform them into a consistent output type without losing type information. We need a way to define a “blueprint” for the transformation that can adapt to the specific types involved. Without generics, we’d be forced to use any types, which would bypass TypeScript’s type checking capabilities, leading to potential runtime errors.

We’ll leverage the following TypeScript features:

  • Generics (<T, U>): To define placeholders for the input and output types. This allows our mapper function to work with different data types while preserving type safety.
  • Interfaces: To define the shape of both the input and output data. This provides a clear structure for our data and enables us to enforce type constraints.
  • Type Aliases: For cleaner syntax and readability, especially when dealing with complex types.

Requirements

Here’s a breakdown of the requirements:

  • Functional:
    • The mapper function should accept an input object and return a transformed output object.
    • The input and output types should be generic, allowing the function to handle different data structures.
    • The transformation logic should be defined by a separate function that takes the input object and returns the transformed output object.
    • The mapper should maintain type safety, ensuring that the input and output types are correctly inferred.
  • Non-Functional:
    • The code should be reusable and easily adaptable to different data structures.
    • The code should be well-documented and easy to understand.
    • The code should be type-safe, minimizing the risk of runtime errors.

Solutions

/**
 * Defines the structure of our simplified user.
 */
interface SimplifiedUser {
  id: number;
  name: string;
  isActive: boolean;
}

/**
 * Defines the structure of a data source user (e.g., from a database or API).
 */
interface DataSourceUser {
  userId: number;
  fullName: string;
  accountStatus: string;
}


/**
 * Type alias for mapping function
 */
type Mapper<T, U> = (input: T) => U;


/**
 * Maps a DataSourceUser to a SimplifiedUser.
 * @param user The DataSourceUser object.
 * @returns A SimplifiedUser object.
 */
function mapDataSourceUserToSimplifiedUser(user: DataSourceUser): SimplifiedUser {
  return {
    id: user.userId,
    name: user.fullName,
    isActive: user.accountStatus === "active",
  };
}


/**
 * Generic data mapper function.  It takes an input object of type T and
 * transforms it into an output object of type U using the provided mapper function.
 * @param input The input object to transform.
 * @param mapper The function that performs the transformation.
 * @returns The transformed output object.
 */
function mapData<T, U>(input: T, mapper: Mapper<T, U>): U {
  return mapper(input);
}

// Assertions for the data mapper
const dataSourceUser: DataSourceUser = {
  userId: 123,
  fullName: "John Doe",
  accountStatus: "active",
};

const simplifiedUser: SimplifiedUser = mapData(dataSourceUser, mapDataSourceUserToSimplifiedUser);

console.assert(simplifiedUser.id === 123, "ID mapping failed");
console.assert(simplifiedUser.name === "John Doe", "Name mapping failed");
console.assert(simplifiedUser.isActive === true, "isActive mapping failed");
  • mapData<T, U>(input: T, mapper: Mapper<T, U>): U: This is our generic mapper function. T represents the type of the input data, and U represents the type of the output data. The mapper argument is a function that takes an input of type T and returns an output of type U. By using generics, we ensure that the mapData function can work with any data type, as long as we provide a corresponding mapper function.
  • Mapper<T,U> = (input: T) => U: Defines a type alias of a Mapper function. This promotes code reusability since it’s basically a blueprint or shape that a function must follow.
  • mapDataSourceUserToSimplifiedUser(user: DataSourceUser): SimplifiedUser: This is a concrete mapper function that takes a DataSourceUser object and transforms it into a SimplifiedUser object. It defines the specific transformation logic for this particular data mapping scenario.
  • console.assert(...): These lines are assertions that check if the mapping was successful. Assertions are a great way to verify that your code is working as expected. If an assertion fails, it will log an error to the console.

Thoughts

We’ve successfully created a reusable data mapper using TypeScript generics. The mapData function is generic and can handle different data types, while the individual mapper functions (like mapDataSourceUserToSimplifiedUser) define the specific transformation logic for each data mapping scenario. This approach provides flexibility, type safety, and reusability.
