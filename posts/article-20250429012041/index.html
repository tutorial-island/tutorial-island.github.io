Implementing a Simple Caesar Cipher in C#

Apr 29, 2025· Tutorial Island

Introduction

Today we are going to implement a simple Caesar cipher in C#. A Caesar cipher is a type of substitution cipher where each letter in the plaintext is replaced by a letter some fixed number of positions down the alphabet. This is a classic introductory problem to cryptography and provides a good exercise in character manipulation and modular arithmetic.

Explanation

The Caesar cipher, while simple, presents a few key challenges:

  1. Character Encoding: We need to work with characters and their underlying numerical representation (ASCII or Unicode). We need to be able to convert between characters and their numerical equivalents and back.
  2. Wrap-Around: When shifting letters, we need to handle the “wrap-around” case. For example, if we shift ‘Z’ by 1, we need to get ‘A’. This involves using modular arithmetic.
  3. Case Sensitivity: We need to decide whether to maintain case or convert all text to a single case (e.g., lowercase). For this example, we will preserve case.
  4. Non-Alphabetic Characters: We need to decide how to handle non-alphabetic characters such as spaces, punctuation, and numbers. In this implementation, we will simply leave these characters untouched.

To tackle these challenges, we’ll use the following C# functionalities:

  • char.IsLetter(char): To check if a character is an alphabet letter.
  • char.IsUpper(char) and char.IsLower(char): To determine if a character is uppercase or lowercase.
  • Type casting between char and int: To access the underlying numerical representation of the characters.
  • The modulo operator %: To handle the wrap-around.

Requirements

  • Functional Requirements:

    • Implement an Encrypt function that takes a string and an integer key (shift value) as input and returns the encrypted string.
    • The Encrypt function should preserve the case of letters.
    • The Encrypt function should leave non-alphabetic characters unchanged.
    • The shift should wrap around the alphabet (e.g., shifting ‘Z’ by 1 should result in ‘A’).

  • Non-Functional Requirements:

    • The code should be clear, readable, and well-commented.
    • The solution should be efficient enough for small to medium-sized strings.

Solutions

using System;
using Microsoft.VisualStudio.TestTools.UnitTesting;

public class CaesarCipher
{
    public static char ShiftChar(char c, int key)
    {
        if (!char.IsLetter(c))
        {
            return c; // Return non-alphabetic characters unchanged.
        }

        char baseChar = char.IsUpper(c) ? 'A' : 'a'; // Determine the base character ('A' for uppercase, 'a' for lowercase)
        int offset = c - baseChar; // Calculate the offset from the base character.
        int shiftedOffset = (offset + key) % 26; // Apply the shift and use modulo to handle wrap-around.
        if (shiftedOffset < 0) shiftedOffset += 26; // Handle negative shift values
        return (char)(baseChar + shiftedOffset); // Convert the shifted offset back to a character.
    }


    [TestMethod]
    public void TestShiftCharPositiveShift()
    {
        Assert.AreEqual('D', ShiftChar('A', 3));
        Assert.AreEqual('a', ShiftChar('x', 3));
    }

    [TestMethod]
    public void TestShiftCharNegativeShift()
    {
        Assert.AreEqual('X', ShiftChar('A', -3));
        Assert.AreEqual('y', ShiftChar('b', -3));
    }

    [TestMethod]
    public void TestShiftCharNonLetter()
    {
        Assert.AreEqual('1', ShiftChar('1', 3));
        Assert.AreEqual(' ', ShiftChar(' ', 3));
    }
}

This code defines a ShiftChar function that takes a character and a key as input and returns the shifted character. It first checks if the character is a letter. If not, it returns the character unchanged. If it is a letter, it determines whether it’s uppercase or lowercase and calculates the offset from the base character (‘A’ or ‘a’). It then applies the shift, using the modulo operator % to handle wrap-around. Finally, it converts the shifted offset back to a character. Negative shift values are properly handled. The test methods ensure proper function using positive and negative shifts, and that non-letter are treated as expected.

    public static string Encrypt(string text, int key)
    {
        string result = "";
        foreach (char c in text)
        {
            result += ShiftChar(c, key); // Apply the shift to each character and append to the result.
        }
        return result;
    }

    [TestMethod]
    public void TestEncryptSimple()
    {
        Assert.AreEqual("Khoor, Zruog!", Encrypt("Hello, World!", 3));
        Assert.AreEqual("Lipps Asvph!", Encrypt("Hello World!", 4));
    }

    [TestMethod]
    public void TestEncryptNegativeShift()
    {
        Assert.AreEqual("Ebiil, Tloia!", Encrypt("Hello, World!", -3));
        Assert.AreEqual("A", Encrypt("D", -3));
    }

This code defines an Encrypt function that takes a string and a key as input and returns the encrypted string. It iterates through each character in the input string, applies the ShiftChar function to shift the character, and appends the shifted character to the result string. The test methods ensure simple shifts and negative shift values provide the correct output.

Thoughts

We’ve successfully implemented a simple Caesar cipher in C#. We addressed the core challenges of character manipulation, wrap-around, and handling non-alphabetic characters. The code is modular, with the ShiftChar function encapsulating the logic for shifting a single character, and the Encrypt function applying this logic to an entire string. The unit tests provide confidence in the correctness of our implementation.
