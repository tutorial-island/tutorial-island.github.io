Building a Simple URL Shortener in Elixir

Apr 25, 2025· Tutorial Island

Introduction

Today, we’re going to build a very simple URL shortener in Elixir. Imagine you have a really long URL, something like https://www.example.com/very/long/path/with/lots/of/parameters?utm_source=newsletter&utm_medium=email&utm_campaign=promotion. We want to create a shorter, more manageable URL that redirects to this original one. We’ll focus on the core logic of generating a short code and associating it with the long URL, without delving into database persistence or web frameworks.

Explanation

The core challenge of a URL shortener lies in generating unique short codes and associating them with their corresponding long URLs. For this example, we’ll focus on generating a short code from a seed.

Here are the key challenges:

  1. Generating Unique Short Codes: We need a function that produces short, unique strings. For simplicity, we will use a very basic approach based on a seed number (which would ideally come from some ID system in a real implementation).

  2. String Manipulation: Elixir provides excellent string manipulation capabilities, which we’ll leverage to transform the seed number into our short code.

We’ll utilize the following Elixir features:

  • Integer Conversion: We’ll convert a seed number into a string.
  • String Manipulation: We’ll use Elixir’s built-in string functions to generate the short code.
  • Assert Statements: We’ll use ExUnit’s assert macro to verify our functions are working as expected.

Requirements

  • Functional Requirement 1: Generate a short code from a seed number.
  • Functional Requirement 2: The generated short code should be a string.
  • Non-Functional Requirement: The short code generation should be relatively simple to understand for beginners.

Solutions

defmodule URLShortener do
  @alphabet "abcdefghijklmnopqrstuvwxyz0123456789"
  @base length(@alphabet)

  # Generates a short code from a seed integer.
  def generate_short_code(seed) do
    do_generate_short_code(seed, "")
  end

  defp do_generate_short_code(0, acc) do
    acc
  end

  defp do_generate_short_code(seed, acc) do
    index = rem(seed, @base)
    char = Enum.at(@alphabet, index)
    new_seed = div(seed, @base)
    do_generate_short_code(new_seed, char <> acc)
  end
end

# Test assertions for the generate_short_code function.
assert URLShortener.generate_short_code(12345) == "dkw"
assert URLShortener.generate_short_code(1) == "b"

Explanation:

This code snippet implements the generate_short_code function.

  • @alphabet: Defines the character set used for generating the short code. This ensures the output is URL-friendly.
  • @base: Defines the base for our encoding (the length of the alphabet).
  • generate_short_code(seed): The public function that takes an integer seed as input and calls the private recursive function do_generate_short_code.
  • do_generate_short_code(0, acc): base case of the recursion. It returns the accumulator which is the short code.
  • do_generate_short_code(seed, acc): This function uses recursion to transform the seed into a short code. It finds the index into the alphabet based on the modulo, retrieve the character, update the seed with the division and concatenate char into the accumulator. It calls itself until seed is 0.
  • assert: The assert statements verify that the function returns the expected short code for specific seed values. This helps ensure that the logic is correct.

Thoughts

We’ve built a very basic URL shortener that demonstrates how to generate a short code from a seed integer using Elixir’s string manipulation capabilities. This example highlights Elixir’s clear syntax and functional programming style. In a real-world application, you would need to add database integration to store the mappings between short codes and long URLs, handle collisions, and implement a web interface to handle URL redirection. However, this simple example provides a solid foundation for understanding the core concepts involved.
