Implementing a Simple Number Checker in C#

May 1, 2025· Tutorial Island

Introduction

Today, we’re going to implement a simple number checker in C#. This program will take a number as input and determine whether it’s positive, negative, or zero. It’s a fundamental exercise that helps illustrate basic control flow in C#.

Explanation

The challenge here lies in understanding and applying conditional logic. We need to evaluate the input number against zero and then execute different code paths based on the outcome of that evaluation.

To solve this, we’ll use the following C# features:

  • if-else if-else statements: These statements allow us to execute different blocks of code based on whether a condition is true or false.
  • Comparison operators: Specifically, we’ll use > (greater than), < (less than), and == (equal to) to compare the input number with zero.
  • Console Output: We’ll use Console.WriteLine to display the result to the user.

Requirements

  • Functional Requirements:
    • The program must accept a numerical input.
    • The program must determine if the input is positive, negative, or zero.
    • The program must output the appropriate message to the console indicating the result.
  • Non-Functional Requirements:
    • The code should be easy to read and understand.
    • The code should be well-commented.

Solutions

using System;
using System.Diagnostics;

public class NumberChecker
{
    public static string CheckNumber(int number)
    {
        // Check if the number is positive.
        if (number > 0)
        {
            return "Positive";
        }
        // Check if the number is negative.
        else if (number < 0)
        {
            return "Negative";
        }
        // If it's not positive or negative, it must be zero.
        else
        {
            return "Zero";
        }
    }
}

Explanation:

This CheckNumber function takes an integer as input and returns a string indicating whether the number is positive, negative, or zero. The if-else if-else structure ensures that only one of the three possible outcomes is returned. We use comparison operators (>, <, and implied ==) to evaluate the number against zero. We return a string instead of printing directly to the console to allow for easier testing and reusability of the function.

public class AssertExample
{
    public static void AssertNumberChecker()
    {
        Debug.Assert(NumberChecker.CheckNumber(5) == "Positive", "Test Case 1 Failed: Positive number check failed.");
        Debug.Assert(NumberChecker.CheckNumber(-5) == "Negative", "Test Case 2 Failed: Negative number check failed.");
        Debug.Assert(NumberChecker.CheckNumber(0) == "Zero", "Test Case 3 Failed: Zero number check failed.");
        Debug.WriteLine("All Assertions Passed!");
    }
}

Explanation:

This AssertNumberChecker function contains three Debug.Assert statements. Each assert statement checks the output of the CheckNumber function against an expected value for a given input. If an assertion fails (i.e., the output does not match the expected value), an AssertionException will be thrown (in debug mode), indicating a problem with the CheckNumber function. We use various test cases (positive, negative, and zero) to thoroughly validate the function. Debug.WriteLine is included to signify successful completion of all asserts.

Thoughts

We successfully created a CheckNumber function that accurately determines if a given number is positive, negative, or zero using if-else if-else statements and comparison operators. The assert statements provide confidence in the correctness of our solution by verifying its behavior across different input values. This example demonstrates the fundamental principles of conditional logic and testing, which are crucial for building more complex programs.

using System;
using System.Diagnostics;

public class NumberChecker
{
    public static string CheckNumber(int number)
    {
        // Check if the number is positive.
        if (number > 0)
        {
            return "Positive";
        }
        // Check if the number is negative.
        else if (number < 0)
        {
            return "Negative";
        }
        // If it's not positive or negative, it must be zero.
        else
        {
            return "Zero";
        }
    }

    public static void Main(string[] args)
    {
        // Get input from the user
        Console.Write("Enter a number: ");
        string input = Console.ReadLine();

        // Try to parse the input as an integer
        if (int.TryParse(input, out int number))
        {
            // Check the number and print the result
            string result = CheckNumber(number);
            Console.WriteLine($"The number is: {result}");
        }
        else
        {
            Console.WriteLine("Invalid input. Please enter a valid number.");
        }

        Debug.Assert(CheckNumber(5) == "Positive", "Test Case 1 Failed: Positive number check failed.");
        Debug.Assert(CheckNumber(-5) == "Negative", "Test Case 2 Failed: Negative number check failed.");
        Debug.Assert(CheckNumber(0) == "Zero", "Test Case 3 Failed: Zero number check failed.");
        Debug.WriteLine("All Assertions Passed!");
    }
}
