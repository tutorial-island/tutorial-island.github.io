Kotlin: Implementing a Basic Data Validation Function

Oct 27, 2023· Tutorial Island

Introduction

Today, we’re going to build a simple data validation function in Kotlin. Imagine you’re receiving user input, and you need to ensure that certain fields, like age, are within a reasonable range. We’ll create a function that validates an age value and returns either the valid age or an error message.

Explanation

The challenge here lies in handling potential errors. A user might enter a non-numeric value or an age that’s outside our allowed range (e.g., negative or excessively large). We need a mechanism to gracefully handle these scenarios without crashing our application.

We’ll be leveraging the following Kotlin features:

  • fun Keyword: To define our validation function.
  • Int Type: To represent the age as an integer.
  • String Type: To represent potential error messages.
  • Result Type: To encapsulate either a successful result (the valid age) or a failure (an error message). Result is a type-safe way to handle potential errors in Kotlin.
  • when Expression: For concisely checking multiple conditions (e.g., age < 0, age > 120).

Requirements

  • Functional Requirement 1: The function must accept an Int representing the age.
  • Functional Requirement 2: The function must return a Result<Int> representing either the validated age or an error message.
  • Functional Requirement 3: The function must return an error message if the age is negative.
  • Functional Requirement 4: The function must return an error message if the age is greater than 120.
  • Non-Functional Requirement 1: The code should be readable and well-documented.
  • Non-Functional Requirement 2: The solution should be efficient and avoid unnecessary computations.

Solutions

/**
 * Validates an age value.
 * Returns a Result containing the valid age or an error message.
 */
fun validateAge(age: Int): Result<Int> {
    return when {
        age < 0 -> Result.failure(IllegalArgumentException("Age cannot be negative."))
        age > 120 -> Result.failure(IllegalArgumentException("Age cannot exceed 120."))
        else -> Result.success(age)
    }
}

fun main() {
    // Test cases
    val validAge = validateAge(30)
    assert(validAge.isSuccess)
    assert(validAge.getOrNull() == 30)

    val negativeAge = validateAge(-5)
    assert(negativeAge.isFailure)
    assert(negativeAge.exceptionOrNull()?.message == "Age cannot be negative.")

    val excessiveAge = validateAge(150)
    assert(excessiveAge.isFailure)
    assert(excessiveAge.exceptionOrNull()?.message == "Age cannot exceed 120.")

    val zeroAge = validateAge(0)
    assert(zeroAge.isSuccess)
    assert(zeroAge.getOrNull() == 0)

    println("All tests passed!")
}

This function, validateAge, checks if the provided age meets our validation criteria. It uses a when expression for concise condition checking. If the age is negative or exceeds 120, it returns a Result.failure containing an IllegalArgumentException with a descriptive error message. Otherwise, it returns a Result.success containing the original age. The main function provides various assert statements to verify the function’s behavior with different inputs. The asserts check both the isSuccess and isFailure states of the Result object, as well as the returned value or exception message.

Thoughts

We’ve successfully implemented a basic data validation function in Kotlin using the Result type for error handling. This approach offers a clean and type-safe way to manage potential errors without resorting to exceptions being thrown and potentially crashing the program. The use of when makes the code highly readable. Remember to adapt the validation rules (age limits, etc.) to fit your specific application requirements.
