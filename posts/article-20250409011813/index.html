Working with Records in Haskell: Building a Simple Product Catalog

Introduction

Today, we’re going to dive into Haskell records by building a simple product catalog. Imagine you’re building an e-commerce site. You need a way to represent products, each with a name, price, and a stock quantity. Haskell’s record syntax provides a clean and structured way to do this. We’ll walk through how to define, create, and manipulate records to manage our product information.

Explanation

The main challenges when working with product information include:

  1. Data Structure Definition: How do we represent a product’s attributes (name, price, quantity) in a structured way? Simple tuples can become unwieldy when dealing with many attributes. We need a named structure.
  2. Creating and Accessing Data: How do we easily create new product instances and access their individual attributes without having to remember the position of each value in a tuple?
  3. Immutability and Updates: Haskell values are immutable. How can we “update” a product’s price or quantity without modifying the original value? We will create a new value.

Haskell’s record syntax addresses these challenges directly. We’ll use the data keyword to define a new data type with named fields. Then, we will leverage record syntax to create and access product information. Finally, we’ll explore record update syntax to create new product records based on existing ones.

Requirements

  • Functional Requirement 1: Define a Product data type with fields for name (String), price (Double), and quantity (Int).
  • Functional Requirement 2: Create a function createProduct that takes a name, price, and quantity and returns a Product record.
  • Functional Requirement 3: Create a function adjustPrice that takes a Product and a new price and returns a new Product with the updated price.
  • Functional Requirement 4: Create a function increaseQuantity that takes a Product and an amount to increase by, and returns a new Product with the adjusted quantity.
  • Non-Functional Requirement: The code should be clear, concise, and well-documented. We will use type signatures extensively.

Solutions

data Product = Product
  { name :: String,
    price :: Double,
    quantity :: Int
  } deriving (Show, Eq)

-- | Create a new product.
createProduct :: String -> Double -> Int -> Product
createProduct n p q = Product {name = n, price = p, quantity = q}

main :: IO ()
main = do
  let product1 = createProduct "Laptop" 1200.0 10
  print product1

This block defines the Product data type with record syntax. name :: String, price :: Double, and quantity :: Int define the fields of the record. deriving (Show, Eq) allows us to print and compare Product values easily. The createProduct function provides a convenient way to construct Product values. The main function tests product creation and display.

-- | Adjust the price of a product, returning a new product.
adjustPrice :: Product -> Double -> Product
adjustPrice product newPrice = product {price = newPrice}

main :: IO ()
main = do
  let product1 = createProduct "Laptop" 1200.0 10
  let product2 = adjustPrice product1 1250.0
  print product2

Here, we define adjustPrice. It takes a Product and a new price. The product {price = newPrice} syntax is record update syntax. It creates a new Product record that is identical to the original product except that the price field is updated to newPrice. Because Haskell variables are immutable, we cannot simply update an already existing product so we instead return a brand new product with the change.

-- | Increase the quantity of a product, returning a new product.
increaseQuantity :: Product -> Int -> Product
increaseQuantity product amount = product {quantity = quantity product + amount}

main :: IO ()
main = do
  let product1 = createProduct "Laptop" 1200.0 10
  let product2 = increaseQuantity product1 5
  print product2

Similarly, increaseQuantity takes a Product and an amount to increase the quantity by. It uses record update syntax to create a new product, where the quantity field is updated by adding the amount to the original quantity. This also exemplifies how one can retrieve fields from a product, using the quantity product syntax, which serves as an accessor to the product’s quantity field.

Thoughts

Let’s put it all together into a single code snippet with assertions.

data Product = Product
  { name :: String,
    price :: Double,
    quantity :: Int
  } deriving (Show, Eq)

-- | Create a new product.
createProduct :: String -> Double -> Int -> Product
createProduct n p q = Product {name = n, price = p, quantity = q}

-- | Adjust the price of a product, returning a new product.
adjustPrice :: Product -> Double -> Product
adjustPrice product newPrice = product {price = newPrice}

-- | Increase the quantity of a product, returning a new product.
increaseQuantity :: Product -> Int -> Product
increaseQuantity product amount = product {quantity = quantity product + amount}

main :: IO ()
main = do
  let product1 = createProduct "Laptop" 1200.0 10
  let product2 = adjustPrice product1 1250.0
  let product3 = increaseQuantity product1 5

  -- Assertions to verify the behavior of our functions
  let expectedProduct1 = Product {name = "Laptop", price = 1200.0, quantity = 10}
  let expectedProduct2 = Product {name = "Laptop", price = 1250.0, quantity = 10}
  let expectedProduct3 = Product {name = "Laptop", price = 1200.0, quantity = 15}

  print $ product1 == expectedProduct1
  print $ product2 == expectedProduct2
  print $ product3 == expectedProduct3

This final code snippet provides a complete example of working with records in Haskell. We defined a Product data type, created functions to create, adjust, and increase quantities of products. The assertions verify that our functions behave as expected, demonstrating the key concepts of record creation, access, and (immutable) updates. By breaking down the problem into smaller, manageable parts, we could easily demonstrate the functionalities Haskell record type provides.
