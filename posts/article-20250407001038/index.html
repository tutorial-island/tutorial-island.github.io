Implementing a Simple Stack in Java

Introduction

Today, we’re going to implement a simple Stack data structure in Java. A stack is a fundamental data structure that follows the Last-In, First-Out (LIFO) principle. Think of it like a stack of plates – you always take the top plate off first. We’ll create a stack that can store integers and includes basic operations like push, pop, peek, and isEmpty.

Explanation

Implementing a stack might seem straightforward, but it presents a few challenges:

  • Underlying Data Structure: We need to choose an appropriate data structure to store the elements of the stack. We can use an ArrayList or a simple array. An ArrayList offers dynamic resizing, which simplifies the implementation, while an array requires us to manage the size manually.
  • Stack Overflow/Underflow: We need to handle potential errors like trying to pop from an empty stack (underflow) or pushing onto a full stack (overflow - if we use a fixed-size array). For simplicity, we’ll focus on handling underflow in this tutorial using exceptions.
  • Maintaining the Top: We need to efficiently keep track of the top of the stack to quickly access the last element added. This often involves an index pointer.

To overcome these challenges, we’ll leverage the following Java features:

  • ArrayList: For storing the stack elements. Its dynamic nature allows us to easily add and remove elements without worrying about fixed sizes (for this implementation. we’re going to use a fixed size array).
  • int[]: For storing the stack elements, inside a simple array
  • int top: An integer variable to keep track of the index of the top element in the stack.
  • Exceptions: To signal stack underflow (trying to pop or peek from an empty stack). We’ll use the built-in EmptyStackException.

Requirements

Here’s a summary of the requirements for our IntStack implementation:

Functional Requirements:

  • push(int item): Adds an integer to the top of the stack.
  • pop(): Removes and returns the integer from the top of the stack. Throws EmptyStackException if the stack is empty.
  • peek(): Returns the integer at the top of the stack without removing it. Throws EmptyStackException if the stack is empty.
  • isEmpty(): Returns true if the stack is empty, false otherwise.

Non-Functional Requirements:

  • The stack should handle integer values.
  • The stack should efficiently perform push, pop, and peek operations.
  • The stack should gracefully handle empty stack scenarios.

Solutions

import java.util.EmptyStackException;

class IntStack {

    private int[] stackArray;
    private int top;
    private int capacity;

    public IntStack(int size) {
        stackArray = new int[size];
        capacity = size;
        top = -1; // Initialize top to -1, indicating an empty stack
    }

    public void push(int item) {
        if (isFull()) {
            throw new IllegalStateException("Stack is full");
        }
        stackArray[++top] = item;
    }

     private boolean isFull() {
        return top == capacity - 1;
    }

    public int pop() {
        if (isEmpty()) {
            throw new EmptyStackException();
        }
        return stackArray[top--];
    }

    public int peek() {
        if (isEmpty()) {
            throw new EmptyStackException();
        }
        return stackArray[top];
    }

    public boolean isEmpty() {
        return top == -1;
    }
}


public class StackExample {

    public static void main(String[] args) {
        // Example usage
        IntStack stack = new IntStack(5); // Initialize with a capacity of 5
        stack.push(10);
        stack.push(20);
        stack.push(30);

        System.out.println("Top element: " + stack.peek()); // Output: 30
        System.out.println("Popped element: " + stack.pop()); // Output: 30
        System.out.println("Top element after pop: " + stack.peek()); // Output: 20

        // Testing with Assertions
        IntStack testStack = new IntStack(3);
        testStack.push(5);
        testStack.push(10);

        assert testStack.peek() == 10 : "Peek should return the top element (10)";
        assert testStack.pop() == 10 : "Pop should return the top element (10)";
        assert testStack.peek() == 5 : "After popping 10, peek should return 5";
        testStack.pop();
        assert testStack.isEmpty() : "Stack should be empty after popping all elements";

        try {
            testStack.pop();
            assert false : "Should have thrown an EmptyStackException"; // This line should not be reached
        } catch (EmptyStackException e) {
            // Expected exception
            System.out.println("EmptyStackException caught successfully!");
        }
    }
}

  • IntStack(int size): This is the constructor. It initializes the stackArray with the given size and sets top to -1, indicating an empty stack. capacity stores the initial size of the stack to check for overflow.

  • push(int item): This method adds an item to the top of the stack. top is incremented before the item is added to the array because top initially points to the index below the first element. Checks for isFull stack

  • pop(): This method removes and returns the top element from the stack. It first checks if the stack is empty. If so, it throws an EmptyStackException. Otherwise, it returns the element at the current top index and then decrements top. The order is important to ensure you return the current top value before you move the top pointer down.

  • peek(): This method returns the top element without removing it. Like pop(), it checks for an empty stack and throws an exception if necessary.

  • isEmpty(): A simple helper method that returns true if top is -1, indicating an empty stack.

Thoughts

We’ve successfully created a basic IntStack class in Java that implements the fundamental stack operations. The key takeaways are understanding the LIFO principle, using top as an index to track the top of the stack, and handling the potential EmptyStackException to prevent crashes. The main method in StackExample shows how to use the Stack class. Assertions are also implemented to test its functionality.

This simple stack implementation provides a foundation for understanding more complex data structures and algorithms that rely on the stack principle. You can extend this implementation further by adding features like dynamic resizing or handling different data types.
