Implementing a Simple Text-Based Adventure Game

Apr 16, 2025· Tutorial Island

Introduction

Today, we’re going to create a rudimentary text-based adventure game. The player will start in a room and will be able to choose directions (north, south, east, west) to move to different rooms. This is a classic introductory programming exercise that demonstrates fundamental concepts like input/output, conditional logic, and functions.

Explanation

Building a text adventure game presents several challenges:

  1. Handling Player Input: We need a way to get input from the player (their chosen direction). This involves using Python’s input() function.

  2. Representing the Game World: We’ll need to represent the different rooms and the connections between them. We can use a dictionary to store this information, where keys are room names and values are dictionaries indicating possible exits.

  3. Conditional Logic: We’ll use if/elif/else statements to determine what happens based on the player’s input. If the player types “north” and there’s a north exit, they move to that room; otherwise, we need to tell them they can’t go that way.

  4. Game Loop: The game needs to continue until the player chooses to quit or reaches a winning condition. We’ll use a while loop for this.

We will leverage the following Python functionalities:

  • input(): To capture user commands.
  • print(): To display text to the user.
  • if/elif/else: To handle game logic based on user input.
  • while: To create the main game loop.
  • dict: To represent the game world (rooms and their connections).
  • str.lower(): To normalize the player input to lowercase for easier processing.
  • break: To exit the game loop when the player quits.

Requirements

Functional Requirements:

  • The game should start in a designated starting room.
  • The player should be able to input directions (north, south, east, west).
  • The game should update the player’s location based on their input, if a valid exit exists in that direction.
  • If the player enters an invalid direction or tries to move in a direction where there is no exit, the game should display an appropriate message.
  • The player should be able to quit the game by typing “quit”.
  • The game world should contain at least 3 interconnected rooms.

Non-Functional Requirements:

  • The code should be well-structured and readable.
  • The code should be commented to explain the logic.

Solutions

def get_player_input():
    """
    Gets the player's input and returns it in lowercase.
    """
    action = input("What do you want to do? (north, south, east, west, quit): ").lower()
    return action

assert get_player_input().islower() == True, "Input should be converted to lower case"
assert type(get_player_input()) == str, "Input should return a string"

This function prompts the user for input and converts it to lowercase using .lower(). This makes the game less sensitive to capitalization. We assert the lower case and that the return is a string.

def move_player(current_room, direction, world):
    """
    Moves the player to a new room based on the given direction.

    Args:
        current_room (str): The name of the current room.
        direction (str): The direction the player wants to move.
        world (dict): A dictionary representing the game world.

    Returns:
        str: The name of the new room if the move is successful, otherwise the current room.
    """
    if direction in world[current_room]:
        return world[current_room][direction]
    else:
        print("You can't go that way.")
        return current_room

world_map = {
    "start": {"north": "hall"},
    "hall": {"south": "start", "east": "kitchen"},
    "kitchen": {"west": "hall"}
}
assert move_player("start", "north", world_map) == "hall", "Player should move to the hall"
assert move_player("start", "south", world_map) == "start", "Player should not move, no exit"

This function takes the current room, the desired direction, and the game world as input. It checks if the specified direction is a valid exit from the current room. If it is, it returns the name of the new room. Otherwise, it prints an error message and returns the current room. We create a simple world and assert the navigation.

def play_game():
    """
    The main function that runs the text adventure game.
    """
    world = {
        "start": {"north": "hall", "description": "You are in a dimly lit entrance room."},
        "hall": {"south": "start", "east": "kitchen", "west": "study", "description": "You are in a long, narrow hall."},
        "kitchen": {"west": "hall", "description": "You are in a messy kitchen."},
        "study": {"east": "hall", "description": "You are in a dusty study filled with books."}
    }
    current_room = "start"

    while True:
        print(world[current_room]["description"])  # Show room description.
        action = get_player_input()

        if action == "quit":
            print("Thanks for playing!")
            break
        else:
            current_room = move_player(current_room, action, world)

play_game()

This function defines the game world as a dictionary and sets the starting room. It then enters a while True loop, which continues until the player quits the game. Inside the loop, it gets the player’s input, checks if they want to quit, and if not, calls the move_player function to update the player’s location. It also prints the room description.

Thoughts

We’ve successfully built a simple text-based adventure game. The get_player_input() function handles player input, the move_player() function manages room transitions, and the play_game() function orchestrates the entire game loop. The game world is represented as a dictionary, allowing for easy expansion and modification. Here’s the complete code:
