Implementing a Simple Calculator in Python

Apr 27, 2025· Tutorial Island

Introduction

Today, we’re going to build a simple calculator in Python. This calculator will take two numbers and an operator as input from the user, perform the corresponding arithmetic operation, and display the result. This exercise will help you solidify your understanding of input/output, basic arithmetic operations, and conditional statements in Python.

Explanation

The main challenges in creating a calculator are:

  1. Taking User Input: We need to receive numbers and the operation from the user. We’ll use the input() function to achieve this. Because input() returns a string, we’ll need to convert the numerical inputs to the appropriate data type (float or int).
  2. Performing Arithmetic Operations: We’ll need to handle the four basic arithmetic operations: addition, subtraction, multiplication, and division. We’ll use the standard +, -, *, and / operators.
  3. Handling Division by Zero: Division by zero is undefined. We need to check for this condition and prevent the program from crashing.
  4. Validating Input: The user might enter invalid inputs (e.g., non-numeric values, unsupported operators). We should gracefully handle these cases, informing the user about the error.

To address these challenges, we will use:

  • input(): To read user input.
  • float(): To convert string input to floating-point numbers, allowing for decimal values.
  • if/elif/else statements: To control the flow of execution based on the operator.
  • Exception handling (try/except): To catch potential errors, like ValueError (if the user enters non-numeric input) or ZeroDivisionError.

Requirements

Here’s a breakdown of the calculator’s requirements:

  • Functional Requirements:
    • Accept two numerical inputs (either integers or decimals).
    • Accept one operator (+, -, *, /).
    • Perform the corresponding arithmetic operation.
    • Display the result.
    • Handle division by zero.
    • Handle invalid operator inputs.
    • Gracefully exit if the input is invalid.
  • Non-Functional Requirements:
    • The code should be readable and well-commented.
    • The program should provide clear error messages to the user.

Solutions

def get_number_input(prompt):
  """
  Prompts the user for numerical input and converts it to a float.
  Handles potential ValueError if the input is not a valid number.
  """
  while True:
    try:
      num = float(input(prompt))
      return num
    except ValueError:
      print("Invalid input. Please enter a valid number.")

# Example usage with assertions
assert isinstance(get_number_input("Enter a number: "), float)
# we can't know the exact number they'll enter, but we know it will be a float

This function, get_number_input, is responsible for safely obtaining numerical input from the user. It uses a while True loop to continuously prompt for input until a valid number (that can be converted to a float) is entered. The try...except block handles the ValueError that occurs if the user enters non-numeric input, displaying an error message and prompting again. This ensures that the program doesn’t crash due to invalid input. The result is returned as a float to handle both integer and decimal values.

def perform_operation(num1, num2, operator):
  """
  Performs the arithmetic operation based on the given operator.
  Handles division by zero errors.
  Returns the result or an error message if the operator is invalid or division by zero occurs.
  """
  if operator == '+':
    return num1 + num2
  elif operator == '-':
    return num1 - num2
  elif operator == '*':
    return num1 * num2
  elif operator == '/':
    if num2 == 0:
      return "Error: Division by zero"
    return num1 / num2
  else:
    return "Error: Invalid operator"

# Example usage with assertions
assert perform_operation(5, 2, '+') == 7
assert perform_operation(10, 2, '/') == 5.0

The perform_operation function takes two numbers and an operator as input. It uses if/elif/else statements to determine which arithmetic operation to perform based on the operator. Critically, it checks for division by zero and returns an appropriate error message. If an invalid operator is provided, it also returns an error message. This function encapsulates the core logic of the calculator, performing the calculations and handling potential errors related to the operations themselves.

Thoughts

Here’s the complete calculator program combining the functions we’ve created:

if __name__ == "__main__":
  num1 = get_number_input("Enter the first number: ")
  num2 = get_number_input("Enter the second number: ")

  operator = input("Enter the operator (+, -, *, /): ")

  result = perform_operation(num1, num2, operator)

  print(result)

  assert get_number_input("Enter a number: ")
  assert perform_operation(5, 2, '+') == 7
  assert perform_operation(10, 2, '/') == 5.0
  print("All tests passed!")

This final script pulls together the get_number_input and perform_operation functions to create a working calculator. The if __name__ == "__main__": block ensures that the calculator logic only runs when the script is executed directly (not when imported as a module). It prompts the user for two numbers and an operator, calls perform_operation to calculate the result, and then prints the result to the console. The assertion statements are strategically placed at the bottom to confirm the basic operations behave as expected.
